# projects/ — 長期的な仕事と成果物の正本

データや成果物を伴う固有プロジェクトを、1プロジェクト1ディレクトリで保存する。

## Knowledge・Skillとの分離

```text
Knowledge = 何が分かっているか（証拠・背景・仮説・反証）→ knowledge/
Skill     = どう分析するか（手順・判定・出力契約）      → skills/
Project   = 何を作り、何を残したか（入力・成果物・記録）→ projects/
```

成果物を Knowledge・Skill・`.tmp/` に残さない。
逆に、プロジェクトから生まれた再利用可能な知見は `knowledge/` へ移す。

## 新しいプロジェクトの作成

`_template/` をコピーして作成し、`PROJECT.md` の各項目を書き換える。

```text
project-name/
├── PROJECT.md          # 入口 — 状態・目的・完了条件・成果物・再現方法
├── inputs/             # このプロジェクトだけで使う入力 — 必要な場合のみ
├── outputs/            # 完成した成果物 — 必要な場合のみ
└── runs/               # 再現に必要な実行記録 — 必要な場合のみ
```

すべてのプロジェクトが全ディレクトリを持つ必要はない。必要なものだけを置く。

## 状態と廃棄

各プロジェクトの状態は `PROJECT.md` 冒頭の `状態:` 行で管理する。値は次の3つのみ。

- `active` — 進行中。
- `paused` — 停止中。再開の可能性があるため削除しない。
- `retired` — 廃止決定済み。削除してよい。

削除はディレクトリごと `git rm` し、理由をコミットメッセージに残す。

```text
chore(retire): drop <project-name> — <理由>
```

アーカイブディレクトリは作らない。ゴミ箱はGit履歴に一本化し、復元も履歴から行う。
