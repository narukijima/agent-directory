# guide — Knowledgeの構造と使い方

人間向けの案内。運用の正本は [KNOWLEDGE.md](../KNOWLEDGE.md)。

## 3層の違い

- `raw/` — 内部で生まれた原記録。ユーザーの原文・判断・仮説・決定。編集・削除しない。
- `research/` — 外部から取得した原資料。内容を変えずに保存し、要約や推論を混ぜない。
- `wiki/` — 原資料を読み解いた再利用可能な知識。訂正・更新はここで行う。

## 保存先の判定

内部で生まれたものは `raw/`、外部から取得したものは `research/`、
それらを読み解いた知識は `wiki/`（1資料なら `sources/`、統合なら `topics/`）へ。

## 役割分担

- **ユーザー** — 原文・判断・資料を提供し、知識の正しさを最終判断する。
- **エージェント** — 分類・保存・ノート化・統合・index/logの更新を行う。

## 知識が増える流れ

```text
原資料が届く → raw/ または research/ に保存
→ sources/ に資料別ノートを作成
→ 複数の知見が揃ったら topics/ に統合
→ index.md に登録、log.md に記録
```

## 実運用との関係

Skill（`skills/`）が「どう分析するか」を持つのに対し、Knowledgeは「何が分かっているか」を持つ。
スキル実行中に根拠が必要になったら、`index.md` から該当ページ・原資料へ遡る。
