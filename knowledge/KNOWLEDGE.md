# KNOWLEDGE.md — Knowledge運用の正本

`AGENTS.md` からKnowledge関連の依頼を受けたら、最初にこのファイルを最後まで読むこと。
このファイルの規約は `knowledge/` 配下のすべての作業に適用される。

## 構造

```text
knowledge/
├── KNOWLEDGE.md       # この正本
├── raw/               # 内部で生まれた不変の原記録
├── research/          # 外部から取得した不変の原資料
└── wiki/              # 原資料を再利用可能な知識へ変換
    ├── index.md       # 全体の入口 — 全ページの登録簿
    ├── guide.md       # 構造と使い方の案内
    ├── log.md         # 時系列履歴
    ├── sources/       # 資料別ノート — 1資料1ページ
    └── topics/        # 統合知識 — 1トピック1ページ
```

## 保存先の判定

新しい情報を受け取ったら、次の順で保存先を決める。

1. **ユーザーや運用の内部で生まれたものか** → `raw/` に保存する。
   例: ユーザーの原文・判断基準・決定・独自仮説・観測記録。
2. **外部から取得したものか** → `research/` に保存する。
   例: 論文・書籍の抜粋・記事・外部の方法論。取得元を必ず記録する。
3. **原資料を読み解いた知識か** → `wiki/` に書く。
   1つの資料に基づくなら `sources/`、複数を統合するなら `topics/`。

## 不変規則

- `raw/` と `research/` の既存ファイルは編集・上書き・削除しない。追記が必要なら新しいファイルを作る。
- 外部資料は内容を変えずに保存する。要約・翻訳・推論を `research/` に混ぜない。
- 訂正はWiki側で行い、原記録はそのまま残す。

## 情報の区別

Wikiページでは、次の4つを混ぜずに書き分けること。

- **原資料の内容** — 出典（ファイルパスとページ・行位置）を付けて記す。
- **ユーザーの見解** — 「ユーザーの判断」と明示する。
- **AIの推論** — 「推論」と明示し、根拠となる出典へリンクする。
- **運用ルール** — 従うべき規則として命令形で書く。

出典のない主張をWikiに書かない。

## Wikiの書き方

- 形式はMarkdown。ページ間の参照はウィキリンク `[[ページ名]]` を使う。
- ファイル名は小文字ケバブケース（例: `capital-allocation.md`）。ページ名とファイル名を一致させる。
- `sources/` は1資料1ページ。書誌・要約・重要な主張・数値・資料の限界・原資料へのリンクを持つ。
- `topics/` は1トピック1ページ。複数の `sources/` ノートや内部経験を統合し、根拠へリンクする。
- 検索基盤・データベース・タグ管理ツールを追加しない。Markdownとウィキリンクだけで管理する。

## index と log の更新

- ページを追加・改名・削除したら、同じ作業の中で `wiki/index.md` を更新する。
  indexには全ページを1行説明付きで登録する。
- Knowledgeを変更したら `wiki/log.md` に1行追記する。形式:

  ```text
  YYYY-MM-DD  種別  対象  要約
  ```

  種別は `ingest`(取り込み) / `query`(再利用価値のある照会) / `lint`(点検) / `migration`(構造変更) / `retire`(廃棄)。
  Wikiページを削除したら `retire` で記録する。`raw/`・`research/` は不変規則のとおり削除対象外。
  過去ログは削除しない。

## 一時ファイル

- 取り込み途中の作業ファイルはリポジトリ直下の `.tmp/` に置く。
- 完了時に、正式な保存先へ移すか削除する。`.tmp/` に成果物を残さない。

## 点検（lint）

定期的、または依頼されたときに次を確認し、結果を `log.md` に記録する。

- `index.md` に未登録のページ、実体のない登録がないか
- リンク切れのウィキリンク・出典リンクがないか
- 出典のない主張、`raw/`・`research/` への混入がないか
